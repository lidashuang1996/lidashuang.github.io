本文需要了解 Socket 和 Java IO 的基础知识。

1. 什么是 Tomcat

Tomcat 服务器是一个免费的开放源代码的 Web 应用服务器，属于轻量级应用服务器，在中小型系统和并发访问用户不是很多的场合下被普遍使用，是开发和调试 JSP 程序的首选。对于一个初学者来说，可以这样认为，当在一台机器上配置好 Apache 服务器，可利用它响应 HTML（标准通用标记语言下的一个应用）页面的访问请求。实际上Tomcat是 Apache 服务器的扩展，但运行时它是独立运行的，所以当你运行 Tomcat 时，它实际上作为一个与 Apache 独立的进程单独运行的。

2. Tomcat 处理请求流程

我们知道 Tomcat 是处理请求的一个 Java 应用。处理请求的流程是：

  1. 读取映射配置文件
  2. 创建 SOCKET（TCP） 服务
  3. 接收浏览器发送过来的 HTTP（TCP） 请求
  4. 根据请求的路径进映射执行对应的方法
  5. 返回数据给浏览器

实现上面的功能我们就可以实现一个超级简单的请求处理的服务了。

3. 编写代码

1. 读取映射文件

我们知道最开始编写 Java Web 的代码的时候，我们通常会在 XML 文件中配置请求映射的接口。也就是请求路径映射到具体的方法。

2. 创建 Service 服务对象

我们知道一个请求发送过来，根据路径映射执行对应的方法。我们通常会让映射的类去继承一个抽象父类，让父类去同一调用执行。

3. HttpRequest 和 HttpResponse

我知道 HttpRequest 是用来读取数据的，HttpResponse是用来返回数据的。那么如何去实现呢？Socket 本质上是流的传输，也就是 InputStream 和 OutputStream。我们可以将 InputStream 转为 HttpRequest ，OutputStream 转为 HttpResponse。
